# World State Autonomous Driving Lab (教学最小闭环版)
（智能驾驶 · 世界状态驱动的深度学习实操代码骨架）

---

## 一、文档定位

本说明为“基础学员 + 讲师经验不足”场景优化版本，目标是完成**最小学习闭环**：

- 开源数据 -> 训练世界模型 -> 推理预测 -> 可视化输出

不追求工程化闭环控制，仅保证“可跑通、可理解、可复现”。

---

## 二、学员最终产出（最小标准）

1. 成功跑通训练脚本并生成模型权重
2. 完成一次推理，输出未来状态预测结果
3. 生成至少 1 张对比可视化图（真实 vs 预测）
4. 提交 1 页简短学习记录（问题/方法/结果/反思）

---

## 三、最小闭环路线图（唯一必学路径）

data/ -> world_model/ -> notebooks/

- data/: 使用 nuScenes mini/sample 数据完成预处理
- world_model/: 训练最简世界模型并预测下一时刻状态
- notebooks/: 可视化对比结果，完成学习总结

> 其他模块（perception / multimodal / control / engineering）为扩展阅读，可在后续进阶使用。

---

## 四、最短运行流程（教学顺序）

1) 下载与准备数据（mini/sample）
2) 运行预处理脚本
3) 训练世界模型
4) 运行推理脚本
5) 打开 notebook 生成可视化结果

---

## 五、跑通标准（完成定义）

- 训练能够正常结束
- 推理输出预测序列
- 至少生成 1 张可视化结果图

---

## 六、常见问题与排查（面向新手）

1) 数据路径错误 -> 检查 data/ 目录配置
2) 依赖缺失 -> 重新安装依赖列表
3) GPU/CPU 不匹配 -> 降低 batch size 或切换 CPU
4) 张量形状报错 -> 检查预处理输出维度
5) 内存不足 -> 缩小 sample 或减少并行
6) 模型权重找不到 -> 检查训练输出路径

---

## 七、讲师辅助话术（简短版）

- 世界状态就是“把复杂场景压缩成可用的数字表示”。
- 世界模型就是“预测这个表示如何随时间变化”。
- 我们今天不做工程化闭环，只做最小可运行学习闭环。
- 跑通 > 追求精度，先理解流程，再谈优化。
